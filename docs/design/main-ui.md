## Key Interaction States

### Wallet Connection Flow

1. **Disconnected**: Show warning badge, "Create Test Wallet" button, and connection input
2. **Connecting**: Show loading spinner, "Connecting..." status
3. **Connected**: Show green checkmark, balance, and scenario controls
4. **Error**: Show error badge, "Retry" button, error message

### Scenario Activation

- **All Connected**: Show full scenario controls and interactions
- **Partially Connected**: Show warning, disable scenario controls, highlight disconnected wallets
- **None Connected**: Show setup instructions, guide user to connect wallets

### Bottom Section Behavior

- Always anchored to bottom of viewport
- Tabs switch between different visualization modes
- Updates in real-time as transactions occur
- Visualization area expands to fill available space
